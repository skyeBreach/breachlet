How issues are created, classified, and tracked in the Breachlet GitLab project.

## Creating Issues

Every issue must include:

- A clear title describing the problem or desired outcome
- A description with enough context for someone unfamiliar with the issue to understand it
- One label from each required group (see Labels below)
- Optional labels where relevant

### Titles

Issue titles should be concise and describe the change, not the task.

- `Add UNIQUE constraint on users.email` not
  `Add a unique constraint to the email column in the users table`
- `Remove axum from domain crate` not
  `We should remove axum because it's unused`

### Descriptions

Descriptions should explain:

1. What the current state is
2. What the desired state is
3. Any relevant context (file paths, error messages, constraints)

### Source

When an issue is created by automation (opencode, CI, scripts), tag it
`source::robot`. Issues created by a person get `source::human`.

## Workflow

Issues follow a simple lifecycle from creation to closure.

1. Issue is created with appropriate labels
2. Issue is assigned to the person responsible for the work
3. Issue receives `status::approved` once reviewed and accepted for work
4. If blocked by another issue or external dependency, apply
   `status::blocked` and link the blocking issue
5. If decided against, apply `status::wontfix` and close the issue with a
   brief explanation
6. When work is complete, close the issue

## Labels

All labels follow the `group::name` format. Group prefixes keep the label
list scannable and prevent ambiguity.

### Required Groups

Every issue must have exactly one label from each required group.

#### Scope

What part of the codebase.

| Label                 | Description                                         | Color     |
| --------------------- | --------------------------------------------------- | --------- |
| `scope::architecture` | DDD layering, dependency direction, crate structure | `#3B82F6` |
| `scope::database`     | Migrations, schema, constraints                     | `#2563EB` |
| `scope::infra`        | Repos, docker, config, justfile                     | `#1D4ED8` |
| `scope::testing`      | Test infra, test repos, coverage                    | `#60A5FA` |
| `scope::meta`         | Docs, descriptions, non-logic cleanup               | `#93C5FD` |

#### Type

Matches conventional commit types.

| Label            | Description                         | Color     |
| ---------------- | ----------------------------------- | --------- |
| `type::bug`      | Fix something broken                | `#F97316` |
| `type::feature`  | New functionality                   | `#FB923C` |
| `type::refactor` | Restructure without behavior change | `#F59E0B` |
| `type::docs`     | Documentation only                  | `#FBBF24` |
| `type::chore`    | Maintenance, cleanup, tooling       | `#D97706` |
| `type::build`    | Dependencies, build system          | `#B45309` |

#### Status

Workflow state.

| Label              | Description                                     | Color     |
| ------------------ | ----------------------------------------------- | --------- |
| `status::approved` | Reviewed and accepted for work                  | `#22C55E` |
| `status::blocked`  | Waiting on another issue or external dependency | `#EF4444` |
| `status::wontfix`  | Decided not to do                               | `#6B7280` |

#### Issue Source

Who created the issue.

| Label           | Description             | Color     |
| --------------- | ----------------------- | --------- |
| `source::robot` | Generated by automation | `#8B5CF6` |
| `source::human` | Created manually        | `#A78BFA` |

### Optional Groups

Apply when relevant. Not required on every issue.

#### Priority

Urgency.

| Label              | Description                            | Color     |
| ------------------ | -------------------------------------- | --------- |
| `priority::high`   | Blocks other work or is time-sensitive | `#DC2626` |
| `priority::medium` | Should be done soon                    | `#F59E0B` |
| `priority::low`    | Nice to have                           | `#3B82F6` |

#### Effort

Estimated size.

| Label            | Description          | Color     |
| ---------------- | -------------------- | --------- |
| `effort::small`  | Under an hour        | `#14B8A6` |
| `effort::medium` | A few hours to a day | `#0D9488` |
| `effort::large`  | Multiple days        | `#0F766E` |

### Color Palette

Groups that aren't status or priority share a cohesive palette:

- **scope**: blue scale (`#3B82F6` to `#93C5FD`)
- **source**: purple scale (`#8B5CF6` to `#A78BFA`)
- **type**: amber/orange scale (`#F97316` to `#B45309`)
- **effort**: teal scale (`#14B8A6` to `#0F766E`)
- **status**: semantic (green = go, red = blocked, gray = dead)
- **priority**: semantic (red = urgent, amber = soon, blue = later)
